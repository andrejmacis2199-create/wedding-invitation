## Инструкция по публикации сайта на Vercel и привязке домена

### 1. Подготовьте проект к Git

Убедитесь, что ваш проект является Git-репозиторием и загружен на GitHub, GitLab или Bitbucket. Next.js проект уже содержит `.gitignore`.

```bash
cd wedding-invitation
git init
git add .
git commit -m "Initial wedding invitation project"
git branch -M main
git remote add origin [ВАША_ССЫЛКА_НА_РЕПОЗИТОРИЙ]
git push -u origin main
```

### 2. Развертывание на Vercel

1.  **Создайте аккаунт Vercel:** Перейдите на [vercel.com](https://vercel.com/) и зарегистрируйтесь (рекомендуется через GitHub).
2.  **Импортируйте проект:** После входа в аккаунт, выберите опцию "New Project" и импортируйте ваш Git-репозиторий (который вы создали на предыдущем шаге).
3.  **Настройки проекта:** Vercel автоматически определит, что это Next.js проект. Вам, скорее всего, не потребуется изменять никакие настройки. Просто нажмите "Deploy".
4.  **Развертывание:** Vercel соберет ваш проект и развернет его. После завершения вы получите ссылку на ваш живой сайт (например, `your-project-name.vercel.app`).

### 3. Привязка собственного домена

1.  **Добавьте домен в Vercel:** В дашборде проекта Vercel перейдите в раздел "Settings" -> "Domains". Введите ваш домен (например, `wedding.ru`).
2.  **Настройте DNS-записи:** Vercel предоставит вам DNS-записи (обычно `A` запись и `CNAME` запись), которые вам нужно будет добавить или изменить у вашего регистратора доменов (там, где вы покупали домен).
    *   **Для корневого домена (например, `wedding.ru`):** Вам нужно будет указать `A` запись, которая будет указывать на IP-адреса Vercel. Vercel предоставит вам эти IP-адреса.
    *   **Для субдомена (например, `www.wedding.ru`):** Вам нужно будет указать `CNAME` запись, которая будет указывать на `cname.vercel-dns.com`.
3.  **Ожидайте обновления DNS:** Обновление DNS-записей может занять от нескольких минут до нескольких часов (иногда до 48 часов). После обновления ваш домен будет указывать на ваш сайт на Vercel.

Вот и все! Ваш свадебный сайт будет доступен по вашему собственному доменному имени.
